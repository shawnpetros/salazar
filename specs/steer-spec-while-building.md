# Steer Feature

ability to steer the feature development by injecting additional context, this triggers a mini replan...if it's EASILY weaved into the remaining/existing feature set, then add it, if not, state that and give options (start over from scratch, wait on this feature because of X, Y, Z, or skip and continue existing work)

## Requirements
- ability to steer mid pass
- validation of feasibility mid-work cycle
- options based on complexity new feature adds to existing spec/feature set

## Acceptance Criteria
- form for mid-work steering
- forked work (mid-work replan/reflow)
- options presented (basically pause for further input)
